## 一、OpenCall Token — 需求分解 & 架构方案

### 1.1 项目概述

OpenCall Token 是一个每日聚合全球艺术公开征集（Open Call）、驻留项目（Residency）与艺术家资助（Grant）的信息平台。目标用户包括艺术家、策展人、艺术学生以及本地小画廊/小艺术家，帮助他们高效发现和跟踪全球范围内的创作机会。

### 1.2 核心需求

#### P0 — 基本浏览（原型已实现）

- 按分类浏览：全部 / 公开征集 / 驻留项目 / 资金资助 / 本地社区
- 卡片式信息展示：标题、机构、截止日期、地点、学科标签、简要描述
- 详情弹窗：完整信息展示 + 外部链接跳转
- 搜索：按标题、机构、地点、描述关键字检索
- 排序：按截止日期（正序/倒序）或最新发布排序
- 自适应布局：桌面端 3 列、平板 2 列、手机 1 列

#### P1 — 社区投稿（原型已实现）

- 本地用户可通过表单提交新的征集/驻留/资助信息
- 提交后数据存储于浏览器 localStorage，在"本地"频道展示
- 提交数据待人工审核后正式发布

#### P2 — 数据爬虫与自动更新（后续阶段）

- 定时爬取全球艺术征集平台（如 Res Artis、Transartists、CaFE 等）和机构官网
- 自动结构化提取：标题、机构、截止日期、地点、学科、资助金额
- 去重与智能合并
- 每日增量更新

#### P3 — 用户系统与个性化（后续阶段）

- 用户注册/登录（邮箱或微信）
- 收藏与关注特定机构/类型
- 截止日期日历提醒（邮件/推送）
- 个人机会推荐

#### P4 — 本地生态扩展（后续阶段）

- 本地小画廊/小艺术家需求发布专区
- 城市/地区筛选
- 中英双语支持

**技术栈（原型阶段）：**

- 纯前端静态网站：HTML5 + CSS3 + Vanilla JavaScript
- 无后端依赖，无需构建工具
- 数据硬编码于 JS 文件，社区投稿存储在浏览器 localStorage
- 部署：任意静态文件托管服务

└──────────────────────────────────────┘

**关键技术决策：**

- **前端框架无锁定**：原型使用原生 JS，后续可渐进式引入 Vue 或 React
- **渐进式增强**：后端按需添加，不重构前端即可接入 API
- **数据优先**：内容质量与覆盖度是核心壁垒，UI 简洁高效即可

### 1.4 数据模型

```json
{
  "id": "string",                     // 唯一标识
  "type": "opencall|residency|grant", // 类型
  "title": "string",                  // 标题
  "organization": "string",           // 主办机构
  "deadline": "YYYY-MM-DD|null",      // 截止日期 (null=全年开放)
  "location": "string",              // 地点
  "disciplines": ["string"],         // 适用学科列表
  "funding": "string",               // 资助说明
  "description": "string",           // 描述
  "url": "string",                   // 原始链接
  "source": "string",                // 来源
  "posted_at": "YYYY-MM-DD",         // 发布日期
  "featured": "boolean",             // 是否精选推荐
  "_local": "boolean?"               // 社区投稿标记 (前端仅用)
}
```

### 1.5 原型状态说明

当前原型已完成 P0 和 P1 的全部功能，包含 25 条全球示例数据（8 条征集 + 8 条驻留 + 9 条资助），覆盖知名机构如惠特尼美术馆、Ars Electronica、MacDowell、DAAD、古根海姆基金会等。社区投稿表单支持完整的 CRUD 流程（写入 localStorage、列表渲染、详情查看）。

## 二、竞品分析

### 2.1 竞品概览

| 竞品 | 类型 | 覆盖范围 | 亮点 | 不足 |
|------|------|----------|------|------|
| **Res Artis** | 驻留数据库 | 全球 | 驻留项目最全，收录 700+ 机构 | 仅驻留，无征集/资助；界面陈旧 |
| **Transartists** | 驻留平台 | 全球 | 信息结构化好，支持筛选 | 同 Res Artis，类型单一 |
| **Call for Entry (CaFE)** | 征集平台 | 美国为主 | 权威性强，与政府机构合作 | 仅限美国，需付费使用 |
| **ArtConnect** | 综合机会平台 | 全球 | 界面现代，包含征集+驻留+资助 | 收费模式（月费 $12+） |
| **artsdeadline.com** | 截止日数据库 | 欧美为主 | 截止日期导向，更新及时 | 设计老旧，无中文 |
| **Curatorspace** | 策展机会平台 | 全球 | 策展方向专业 | 面向策展人，范围窄 |
| **Akoun** | 聚合推送 | 全球 | 每日更新，邮件推送 | 付费订阅（年费 €49+） |

### 2.2 OpenCall Token 差异化策略

**核心差异化：**

1. **全类型覆盖**：同时覆盖公开征集、驻留项目和资金资助三大类，一站式服务
2. **本地化社区**：专设"本地"频道，填补中小城市/本地画廊需求的信息空白
3. **免费开放**：对标 ArtConnect/Akoun 的付费模式，基础服务完全免费
4. **中文原生**：面向中文艺术社区，降低语言门槛
5. **渐进式功能**：从轻量原型起步，逐步增强而非一开始就重投入

**目标用户群：**

- 国内独立艺术家、艺术专业学生
- 海外华人艺术家
- 本地小画廊主、策展人
- 艺术管理从业者

### 2.3 竞争壁垒构建

- **数据壁垒**：建立自有的爬虫与聚合管道，覆盖中文+英文+日韩等多语言来源
- **社区壁垒**：本地投稿+人工审核机制，形成用户共建的差异化数据源
- **体验壁垒**：简洁高效的浏览体验 vs 竞品普遍过时的 UI

## 三、部署方案

### 3.1 原型阶段部署（当前）

项目为纯静态网页，无需后端服务，多种免费托管方案可选：

| 方案 | 优点 | 适用场景 |
|------|------|----------|
| **GitHub Pages** | 免费，支持自定义域名，HTTPS | 个人项目，展示原型 |
| **Netlify** | 免费套餐（100GB/月），自动 CI/CD | 团队协作，快速迭代 |
| **Vercel** | 免费套餐，全球 CDN，自动 HTTPS | 追求性能，后续可能引入前端框架 |
| **阿里云 OSS + CDN** | 国内访问速度快 | 主要面向国内用户 |

**推荐方案（原型）：Netlify**

- 自动从 Git 仓库部署
- 免费 TLS/HTTPS
- 支持自定义域名
- 操作极简：连接仓库即可上线

部署步骤：

1. 将 `outputs/opencall-token/` 目录下所有文件推送到 GitHub 仓库
2. 登录 Netlify → New site from Git → 选择仓库
3. 构建命令留空（纯静态），发布目录设为 `outputs/opencall-token/`
4. 可选：绑定自定义域名，开启 HTTPS

### 3.2 后端引入阶段部署

当项目引入后端 API（Node.js/Express 或 Python/FastAPI）和数据库后：

#### 轻量方案（推荐起步）

```
前端: Netlify / Vercel (全球 CDN)
后端: Railway / Render / Fly.io (自动扩缩容)
数据库: Supabase (免费 PostgreSQL) / MongoDB Atlas (免费 512MB)
缓存: Upstash Redis (免费套餐)
```

- 月成本估算：$0-5（在免费套餐范围内）
- 自动 HTTPS，自动部署

#### 国内部署方案

```
前端: 阿里云 OSS + CDN
后端: 阿里云 ECS (轻量应用服务器，¥24/月起)
数据库: 阿里云 RDS PostgreSQL / 腾讯云 MySQL
域名: 阿里云/腾讯云域名 + ICP 备案
```

### 3.3 爬虫与数据管道部署

**组件与部署：**

- **定时爬虫**：Python Scrapy/Playwright，部署于 GitHub Actions（免费）或阿里云函数计算
- **任务调度**：Celery Beat 或 AWS EventBridge / 阿里云 SchedulerX
- **数据清洗**：Python 脚本，包含去重、分类、自动标签

**工作流示意：**

```
定时触发器 (Cron) → 爬虫任务 → 数据清洗/去重 → PostgreSQL → API 服务 → 前端展示
```

### 3.4 CI/CD 流程

```
Git Push → GitHub Actions / Netlify CI
    ├─ 代码检查 (ESLint)
    ├─ 单元测试 (Vitest / Jest)
    ├─ 构建 (可选：Vite/Rollup)
    └─ 部署 (自动推送到托管平台)
```

### 3.5 成本估算总结

| 阶段 | 月成本 | 说明 |
|------|--------|------|
| 原型（纯静态） | $0 | Netlify/GitHub Pages 免费套餐 |
| 引入后端 API | $0-5 | Free tier 足以支撑小规模用户 |
| 引入爬虫 + 数据库 | $5-20 | 需付费数据库或轻量云服务器 |
| 规模化运营 | $50-200+ | 多台服务器 + 托管数据库 + CDN |

## 四、开发路线

### Step 1 — 原型验证（当前已完成）

**目标**：跑通核心信息流，验证需求，获取首批用户反馈

- 纯静态前端（HTML + CSS + Vanilla JS）
- 25 条硬编码全球示例数据，覆盖公开征集 / 驻留 / 资助三大类型
- 分类浏览、搜索、排序、响应式布局
- 社区投稿表单 + localStorage 存储
- 部署于免费静态托管（Netlify / GitHub Pages）
- 上线后手动维护真实数据，测试用户使用习惯

**交付物**：可公开访问的聚合页原型

---

### Step 2 — 后端与数据管道搭建

**目标**：告别硬编码，建立自动化数据流水线

**2a. 后端 API 服务**

- 搭建轻量后端（Node.js/Express 或 Python/FastAPI）
- 设计 RESTful API：GET /api/opportunities、POST /api/opportunities、GET /api/opportunities/:id
- 前端从 localStorage 数据源迁移到 API 请求
- API 层做缓存（Redis），降低数据库压力

**2b. 数据库**

- PostgreSQL 存储结构化机会数据
- 使用 Supabase（免费 PostgreSQL）或 Railway 托管

**2c. 爬虫管道（第一阶段）**

- 选定 3-5 个高价值源站作为首批爬虫目标（如 Res Artis、Transartists、CaFE）
- Python Scrapy 实现定向爬虫 + 结构化提取
- 部署于 GitHub Actions，每日定时触发（Cron）
- 数据清洗、去重、自动分类后写入 PostgreSQL

**交付物**：API 在线 + 至少 3 个源站自动爬取 + 每日增量更新

---

### Step 3 — 体验升级与用户系统

**目标**：从信息工具进阶为个性化服务平台

**3a. 前端升级**

- 渐进式引入前端框架（Vue 3 或 React）
- 优化加载性能：图片懒加载、虚拟滚动、骨架屏
- 移动端体验打磨：PWA 支持、离线浏览

**3b. 用户系统**

- 邮箱 / 微信登录（Auth0 或自建 JWT）
- 收藏机会、关注机构
- 截止日期日历提醒（邮件提醒 + 浏览器推送）
- 个人机会推荐（基于收藏历史、学科偏好）

**3c. 社区审核机制**

- 社区投稿后台审核面板
- 审核通过后自动进入主数据池
- 防重复提交 + 垃圾信息过滤

**交付物**：带用户系统的完整 Web 应用，支持个性化订阅

---

### Step 4 — 爬虫规模化与多语言覆盖

**目标**：建立数据壁垒，覆盖更多源站和语种

- 爬虫目标扩展至 20+ 源站，覆盖英文、中文、日文、韩文等
- 接入 LLM（GPT-4 / 本地模型）进行非结构化页面解析
- 自动翻译标题和描述（中英对照）
- 数据质量监控面板：爬取成功率、数据完整性、时效性
- 开放 API 供第三方接入（可选）

**交付物**：日均更新 50+ 条机会，覆盖全球主要艺术机会源

---

### Step 5 — 本地生态与商业化探索

**目标**：从全球聚合延伸至本地服务，探索可持续模式

**5a. 本地生态**

- 本地画廊 / 小艺术家需求发布专区
- 城市 / 地区维度筛选与地图模式
- 机构主页：展示机构所有历史机会 + 关注度数据

**5b. 商业化探索**

- 机构付费推广（置顶机会、品牌曝光）
- 招聘 / 展览信息增值服务
- 艺术家作品集展示专区

**5c. 国际化**

- 完整中英双语界面切换
- 面向海外华人艺术社区的推广

**交付物**：兼具全球数据覆盖与本地社区粘性的艺术机会平台
